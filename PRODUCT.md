# Product

## Register

brand

## Users

Hydro-Minim 面向 Halo 博客站点作者、独立创作者和主题二次开发者。访问者通常在阅读文章、浏览分类归档、查看作者与扩展页面时接触主题；站点作者则通过 Halo Console 配置导航、页脚、配色、搜索和快捷入口。

## Product Purpose

Hydro-Minim 是“氢”系列的首款 Halo 主题，以轻盈动效和极简视觉服务 Halo 原生 Thymeleaf + Vite 架构。成功标准是：站点看起来克制、有识别度、运行稳定，并且不会因为平台数据、插件缺失或移动端环境而破坏阅读体验。

## Brand Personality

轻、克制、结构化。界面应像低饱和的纸面实验品：灰白背景、细线、Space Mono、轻颗粒、温和动效和少量强调色共同构成“氢·简”的记忆点。

## Visual Boundaries

不要变成通用博客模板、信息过载布局、纯黑高对比仪表盘、霓虹科技风、高装饰视觉或堆满卡片的营销页。不要引入 React、Radix、shadcn 运行时。

## Design Principles

1. 保持氢感：轻、薄、透、低饱和，所有视觉增强都要服务于这种气质。
2. 先尊重平台：模板表达式、Finder 数据、插件可用性和后台配置要有空值兜底。
3. 互动要有边界：导航收缩、主题光幕、Hero reveal、卡片 hover 等状态机彼此独立。
4. 内容优先：文章、分类、归档、搜索和扩展页要各有结构，不把所有页面压成一种卡片网格。
5. 源码可维护：只编辑源文件，保持 Thymeleaf、TypeScript 和 CSS 的职责清晰。

## Accessibility & Inclusion

保留键盘焦点、语义结构、响应式布局、深浅色模式和 `prefers-reduced-motion` 兼容。暂无单独 WCAG 等级承诺，后续改动默认以 WCAG AA 作为可读性与对比度底线。
